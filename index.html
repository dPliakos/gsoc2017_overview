Abstract

Treematcher is a subproject of ete3. Target of treematcher is to perform search to phylogenetic trees using tree patterns that allows relax connection between tree nodes in newick format mimicking the regular expressions' functionality.

Targets:

  • Have the ability to use regular expressions' symbols (metacharacters) and functionality to describe a pattern
  • Allow treematcher's pattern describe loose connection between nodes
  • Allow treematcher's pattern describe patterns in a node's descendants
  • Treematcher's patterns describe attributes to so test the presence of conditions in a tree nodes

Background

Treematcher is not a new project. The base code was already written the time GSoC started and that code was used to extend it's functionality. The treematcher could match patterns that wheren't use metacharacters and could skip nodes using the "one or more" (intermediate nodes) between two nodes functionality ( the + sumbol).

Coding periods

First Coding Period

So in the first coding period the was about understaning the logic and the approach of the problem and defining how treematcher will adopt regular expressions' functionality.
That time was about the relax connection between the nodes. The basic symbols' functionality was implemented (like *, +, ^, $).
At late first period the implementation logic changed and insted of treating every metacharacter indipendently, they where categorized in cases.

Second Coding Period

In the second coding period the rest of the symbols and some sets where implemented. That was done quickly thanks to the new technique that was introduced.
The use of purly python expressions to define a node gave the ability for more complex patterns to be expressed like numerical comparisons between sets or the whole tree.
In the second coding period the ability to describe a pattern about a node's descendants using metacharacters was introduced and proved that the existing algorithm could not satisfy the treematcher needs.
That is an usefull ability that treeematcher should have, so after a long debugging time of the approaches that were tried, the dessision to refactor the code was made and started.

Third coding period

During the third coding period the code was refactored quickly thanks to mentrors' effort, the experience that where gained and the techniques that where learnt.
Before the refactoring the focus was on patterns that describe a tree's topology, but after the refactor the focus is on nodes' descendants patterns resulting the loss of a lot of functionality.
The new approach differs a lot in both implementation and logic to describe the patterns.
That time the command line toolkit was updated and extended.
Treematcher's preperation for mering to it's parent project (ete3) was made and documentation was updated.

Interesting parts

  • The catigorization of metacharacters (period 1)
  • The ability to use pure python expressions to define a node (period 2)
  • Limitations are met at describing nodes' descendants relax patterns (period2)
  • The code reefactoring (period 3)

Conclusion

What is done
Treematcher is able to describe relax connection between nodes.
Treematcher is able to describe relax patterns at a node's descendants.
Treematcher is able to test complex patterns using node's attributes.
